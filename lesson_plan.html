Lesson 2

การทำงานของหน้าเว็บแบบ Single HTML

การทำงานของหน้าเว็บแบบ Single HTML (ซึ่งรวมทั้ง HTML, CSS และ JavaScript ไว้ในไฟล์เดียว) สามารถเปรียบเทียบได้กับการสร้าง "มนุษย์" หรือ "รถยนต์" 1 คัน โดยทั้ง 3 ส่วนนี้มีหน้าที่แตกต่างกัน แต่ต้องทำงานร่วมกันอย่างใกล้ชิด

1. บทบาทและหน้าที่ของแต่ละส่วน

HTML (HyperText Markup Language) - "โครงกระดูกและอวัยวะ"

หน้าที่: กำหนดโครงสร้างและเนื้อหาทั้งหมดของหน้าเว็บ (อะไรอยู่ตรงไหน) เช่น ปุ่มกด, ช่องกรอกข้อมูล (Input), หัวข้อ (Header), หรือตารางแสดงผล

เปรียบเทียบ: เหมือนโครงเหล็กของรถยนต์ หรือโครงกระดูกของมนุษย์ที่ระบุว่าตรงนี้คือแขน ตรงนี้คือขา

CSS (Cascading Style Sheets) - "ผิวหนัง เสื้อผ้า และหน้าตา"

หน้าที่: ตกแต่งหน้าตา จัดวางตำแหน่ง สีสัน ขนาดอักษร และความสวยงาม (ทำให้อวัยวะดูดีขึ้น)

เปรียบเทียบ: เหมือนสีพ่นรถยนต์ เบาะหนัง หรือเสื้อผ้าเครื่องแต่งกายของมนุษย์ที่ทำให้โครงสร้างดูสวยงามน่ามอง

JavaScript (JS) - "ระบบประสาทและกล้ามเนื้อ"

หน้าที่: ควบคุมพฤติกรรม ตรรกะ การคำนวณ และการตอบสนองแบบเรียลไทม์ (ทำให้ขยับและคิดได้) เช่น เมื่อคลิกปุ่มแล้วให้เริ่มคำนวณอัลกอริทึม MCTS ทันที โดยไม่ต้องดาวน์โหลดหน้าเว็บใหม่

เปรียบเทียบ: เหมือนเครื่องยนต์และระบบสมองกลของรถยนต์ หรือระบบประสาทของมนุษย์ที่สั่งการให้ก้าวขาเดิน

Lesson 2

2. แผนผังและขั้นตอนการทำงานร่วมกันในเบราว์เซอร์

เมื่อคุณเปิดไฟล์ solar.html ขึ้นมาบนเบราว์เซอร์ (เช่น Chrome, Safari, Edge) คอมพิวเตอร์จะประมวลผลตามขั้นตอนเหล่านี้:

1

การโหลดโครงสร้าง (Parsing HTML)

เบราว์เซอร์จะอ่านไฟล์จากบนลงล่าง เจอแท็ก เช่น <div>, <input>, <button> มันจะสร้างสิ่งที่เรียกว่า DOM (Document Object Model) ซึ่งเป็นต้นไม้โครงสร้างของวัตถุทั้งหมดบนหน้าจอ

2

การแต่งตัว (Applying CSS)

เมื่ออ่านไปเจอแท็ก <style> หรือคลาสของ Tailwind CSS (เช่น bg-white p-5 rounded-xl) เบราว์เซอร์จะนำสไตล์เหล่านี้ไปครอบลงบนโครงสร้าง HTML ทำให้ปุ่มกลายเป็นสีเหลืองอมส้ม แผงแสดงผลกลายเป็นบล็อกมนๆ สวยงาม (สร้าง Render Tree -> แสดงผลบนจอ)

3

การเปิดระบบควบคุม (Executing JavaScript)

เมื่ออ่านลงมาถึงแท็ก <script> เบราว์เซอร์จะเตรียมเปิดระบบประมวลผลของ JavaScript ขึ้นมาทำงานในพื้นหลัง โดยจะทำการสร้างฐานข้อมูลจำลอง (tempDB), ตั้งค่าปุ่มกดให้รอรับแรงคลิก (onclick), และเตรียมอัลกอริทึมคำนวณ (MCTS) ให้พร้อมใช้งานเพื่อรอรับเหตุการณ์ (Event)

3. ตัวอย่างการทำงานร่วมกันในโปรแกรมโซลาร์เซลล์ของคุณ

วงจรรูปแบบการทำงานจริงเมื่อผู้ใช้เปิดใช้งานระบบเลือกอุปกรณ์โซลาร์เซลล์:

ขั้นที่ 1

หน้าตาและการป้อนข้อมูล (HTML + CSS)

ผู้ใช้เห็นช่องกรอกข้อมูล "ความต้องการไฟฟ้า (Wh/วัน)" เป็นกล่องข้อความสีขาวขอบเทาอย่างสวยงามพึงประสงค์ นี่คือหน้าที่ของ HTML ที่สร้างช่องกรอก และ CSS ที่จัดให้มันดูเรียบร้อย

ขั้นที่ 2

การเกิดเหตุการณ์ (Event Trigger)

เมื่อผู้ใช้กรอกตัวเลขแล้วใช้นิ้วคลิกหรือกดปุ่ม "เริ่มการวิเคราะห์ MCTS" เหตุการณ์ที่เรียกว่า click จะเกิดขึ้น ปุ่มนี้จะส่งสัญญาณไปเรียกฟังก์ชัน runMCTSOptimization() ใน JavaScript มารับไม้ต่อ

ขั้นที่ 3

การประมวลผลแอบซ่อน (JavaScript Logic)

JavaScript จะวิ่งไปหยิบตัวเลขที่ผู้ใช้กรอกมาจาก HTML ดึงข้อมูลอุปกรณ์จากฐานข้อมูลชั่วคราว (tempDB) แล้วสั่งรันลูปสุ่มคำนวณสเปคและ NPV อย่างรวดเร็วในหน่วยความจำ (RAM) โดยที่ผู้ใช้จะไม่เห็นสูตรคณิตศาสตร์เหล่านี้บนหน้าจอเลย

ขั้นที่ 4

การเปลี่ยนสภาพหน้าจอ (Dynamic UI Update)

เมื่อ JavaScript คำนวณเสร็จมันจะหันกลับไปสั่ง HTML ว่า ให้เอาชื่อรุ่นแผงนี้ไปใส่ในแท็ก ID ชื่อ resPanelModel และสั่งแก้สไตล์ CSS ให้ข้อความกระพริบตัวโหลด (loadingStatus) หายไป แล้วเปิดแสดงกล่องผลลัพธ์ (resultContainer)

ด้วยเหตุนี้ Single HTML จึงเป็นโครงสร้างที่ทรงพลังมากสำหรับการสร้างแอปพลิเคชันขนาดเล็กถึงปานกลาง เพราะไฟล์เดียวบรรจุทั้งภาพลักษณ์ (HTML/CSS) และสมองกล (JavaScript) ทำให้มันทำงานร่วมกันได้อย่างรวดเร็ว ไหลลื่น โดยไม่ต้องพึ่งพาเซิร์ฟเวอร์ภายนอกเลยครับ

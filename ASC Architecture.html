1. สถาปัตยกรรมระบบ (System Overview)

Auto Solar Copilot เป็นเครื่องมือซอฟต์แวร์วิศวกรรมบนเว็บ (Web-based Engineering Tool) ที่ถูกออกแบบมาเพื่อการออกแบบระบบผลิตไฟฟ้าจากพลังงานแสงอาทิตย์บนหลังคา (Solar Rooftop) และระบบติดตั้งบนพื้นดิน

ระบบไม่ได้เป็นเพียงเครื่องมือวาดแผนที่ (Drawing Tool) แต่ผสานการทำงานของระบบสารสนเทศภูมิศาสตร์ (GIS) ผ่านไลบรารี Leaflet เข้ากับเอนจินคณิตศาสตร์วิศวกรรมไฟฟ้าแบบเรียลไทม์ ทำให้ผู้ใช้สามารถประเมินพื้นที่จริง วาดแผงโซลาร์เซลล์ และตรวจสอบความปลอดภัยทางไฟฟ้าได้ในหน้าจอเดียว

จุดเด่นของสถาปัตยกรรม (Architectural Highlights)

  • Hybrid Web-GIS: ใช้ Leaflet (JavaScript Library แบบ Open-source สำหรับการพัฒนาแผนที่แบบโต้ตอบ (Interactive Web Map)) นำพิกัดตำแหน่งของอาคาร,พื้นที่ มาวาดภาพจำลอง (Polygon Rendering) และจัดการด้วย SVG/Canvas เพื่อให้แผนที่คงสัดส่วนเดิม
  • EIT Sizing Engine: ฝัง (รหัสเทียม (Pseudocode))มาตรฐานวิศวกรรมสถานแห่งประเทศไทย (วสท.) ไว้ในระบบคณิตศาสตร์ของโปรแกรม เพื่อควบคุมแรงดันกระแสตรงไม่ให้เกินขีดจำกัด
  • Zero-Loss Data Binding: โครงสร้าง Database รูปแบบ JSON (DB1 และ DB2) ที่ทำให้สามารถทำเข้า-ส่งออก (Import/Export) โปรเจกต์ได้ 100% โดยที่พารามิเตอร์ไม่สูญหาย วางแผนเมื่อต้องนำระบบไปพัฒนาต่อในรูปแบบ Client-Server

2. แบบจำลองด้านพลังงาน (Energy Models & Equations)

การคำนวณในโปรแกรมนี้ถูกออกแบบโดยอ้างอิงมาตรฐานวิศวกรรมสากล (IEC 62548) และมาตรฐาน วสท. ของประเทศไทย เพื่อป้องกันความเสียหายแก่อินเวอร์เตอร์และลดความเสี่ยงการเกิดเพลิงไหม้

2.1 แบบจำลองการจัดสตริง (String Sizing Model)

การหาจำนวนแผงสูงสุดและต่ำสุดต่อสตริง โปรแกรมไม่ได้ใช้แรงดันสภาวะ STC (25°C) มาหารตรงๆ แต่ใช้ ค่าสัมประสิทธิ์อุณหภูมิ (Temperature Coefficient) ในการจำลองแรงดันที่เปลี่ยนแปลงไปตามสภาพอากาศของประเทศไทย:

// สมการหาจำนวนแผงสูงสุด (อุณหภูมิต่ำ T_min = 15°C)

VOC_cold = VOC_STC × [ 1 + ( γv / 100 ) × (Tmin - 25) ]

Max_Panels = Floor( Vmax_system / VOC_cold )
// สมการหาจำนวนแผงต่ำสุด (อุณหภูมิสูง T_max = 70°C)

VMP_hot = VMP_STC × [ 1 + ( γv / 100 ) × (Tmax - 25) ]

Min_Panels = Ceil( Vmin_MPPT / VMP_hot )

2.2 แบบจำลอง Optimizer Voltage Clamping

เมื่อระบบตรวจพบการใช้งาน Power Optimizer โปรแกรมจะข้ามการประเมินอุณหภูมิแผง และเปลี่ยนไปใช้เงื่อนไขคุมแรงดันและกำลังไฟฟ้าของ Optimizer (Voltage Clamping) ทำให้สามารถต่อสตริงได้ยาวขึ้น

Max_Panels = Min( Floor(Vmax_INV / Vmax_OPT_output) , Floor(Pmax_StringLimit / Pmax_PV) )

2.3 แบบจำลองอุณหภูมิแผงโซลาร์เซลล์ (Sandia Model โดย David L. King)

โปรแกรมใช้โมเดลของห้องปฏิบัติการ Sandia National Laboratories เพื่อประเมินอุณหภูมิแผงจริง (Tcell) ซึ่งคำนึงถึงความเข้มแสงและความเร็วลม ทำให้ประเมินการสูญเสียในสภาวะการทำงานปกติได้แม่นยำกว่า

Tcell = Tamb + GHI × e(a + b × WS)

แบบจำลองประสิทธิภาพไฟฟ้า (Sandia Array Performance Model: SAPM): คำนวณหาค่ากระแสไฟ (Short-circuit current) และแรงดันไฟฟ้า (Open-circuit voltage) (Tcell) คำนวณหาค่ากระแสไฟ (Short-circuit current) และแรงดันไฟฟ้า (Open-circuit voltage) ออกมาเป็นจุดทำงานต่าง ๆ บนกราฟ I-V Curve เพื่อหาพลังงานไฟฟ้ากระแสตรง (DC Power) ที่แผงผลิตได้

3. ข้อมูลสภาพแวดล้อมและ API (Meteo Data APIs)

ข้อมูลสภาพแวดล้อม (Meteo Data) เป็นหัวใจสำคัญในการจำลองการผลิตไฟฟ้า (Energy Yield) โปรแกรมออกแบบสถาปัตยกรรมให้รองรับ API 3 แหล่ง ซึ่งเป็นมาตรฐานอุตสาหกรรมสากล:

แหล่งข้อมูล API ความละเอียด เหตุผลในการเลือกนำมาใช้งาน
PVGIS (JRC Europe) รายชั่วโมง (TMY) เป็น API ที่น่าเชื่อถือระดับโลก ใช้งานได้ฟรี 100% ไม่มี API Key ถูกตั้งเป็น Default Engine
NASA POWER รายชั่วโมง (Satellite) ครอบคลุมพื้นที่ทั่วโลกแบบ 100% แม้อยู่ในพื้นที่ห่างไกลหรือกลางทะเล
Meteonorm รายชั่วโมง (Interpolation) ถือเป็นมาตรฐานทองคำ (Gold Standard) ในการออกเอกสารกู้เงินธนาคาร

ชื่อโปรแกรม ประเภทสิทธิการใช้ การวิเคราะห์และเปรียบเทียบ

5. แบบจำลองทางการเงิน (Financial & ROI Model)

ในการอัปเกรดจากเครื่องมือทางวิศวกรรมสู่ "เครื่องมือประเมินการลงทุน" ระบบสถาปัตยกรรมจะถูกขยายให้รองรับการคำนวณด้านกระแสเงินสด (Cash Flow) โดยมีสมการชี้วัดทางการเงิน ดังนี้:

5.1 โครงสร้างต้นทุนและรายได้

  • CAPEX (ต้นทุนเริ่มต้น): คำนวณจาก Total DC Capacity (kWp) × ราคาต่อวัตต์
  • OPEX (ต้นทุนดำเนินงาน): ครอบคลุมค่า O&M (เช่น 1% ของ CAPEX/ปี) และค่า Inverter Replacement ในปีที่ 10
  • Revenue (รายได้/ส่วนประหยัด): แบ่งสัดส่วนระหว่าง Self-Consumption (ทดแทนการซื้อไฟ) และ Grid Export (ขายไฟ)

5.2 ดัชนีชี้วัดเพื่อการลงทุน (Financial Metrics)

1. มูลค่าปัจจุบันสุทธิ (Net Present Value : NPV)

เมทริกซ์หลักที่ใช้บอกว่าโครงการคุ้มค่าการลงทุนหรือไม่ในช่วงระยะเวลา 25 ปี

NPV = Σ [ (Rt - Ct) / (1 + d)t ] - CAPEX

2. อัตราผลตอบแทนภายใน (Internal Rate of Return : IRR)

ใช้อัลกอริทึม (เช่น Newton-Raphson) ในการค้นหาเปอร์เซ็นต์ IRR ที่ทำให้ NPV มีค่าเท่ากับ 0

3. ต้นทุนการผลิตไฟฟ้าต่อหน่วย (Levelized Cost of Energy : LCOE)

ใช้เปรียบเทียบว่าต้นทุนการสร้างโรงไฟฟ้าตกอยู่ที่กี่บาทต่อ 1 หน่วยไฟฟ้า (บาท/kWh)

LCOE = [ CAPEX + Σ (OPEXt / (1 + d)t) ] / Σ (Et / (1 + d)t)

6. โครงสร้างสถาปัตยกรรมระบบ (Direct-to-Memory)

Auto Solar Copilot ถูกพัฒนาด้วยแนวคิด "Direct-to-Memory (Client-side Simulation)" ซึ่งประมวลผลทุกอย่างบนเบราว์เซอร์ของผู้ใช้ 100% โดยไม่ต้องพึ่งพาเซิร์ฟเวอร์ (Zero Backend Cost) ส่งผลให้มีความปลอดภัยของข้อมูลสูง (Data Privacy) และประมวลผลได้รวดเร็ว

💡 กระบวนการทำงานหลัก (Data Pipeline)

  • Data Ingestion: นำเข้าไฟล์พารามิเตอร์ (.txt/.json) ผ่าน FileReader เข้าสู่หน่วยความจำ (RAM) โดยตรง
  • Regex Sweeper: ใช้ Regular Expression ระดับสูงในการตัดขยะ (เช่น mapGeoJSON ที่ทำให้เครื่องค้าง) และสกัดเฉพาะพารามิเตอร์ของอุปกรณ์
  • DB3 Memory State: จัดเก็บข้อมูลลงโครงสร้างอ็อบเจกต์ใน sessionStorage ทำให้สามารถสลับหน้าจอ (UI Tabs) ไปมาได้โดยข้อมูลไม่สูญหาย
  • Simulation Engine: วิ่งลูปคำนวณ 8,760 ชั่วโมง (Hourly Step) และระดับเดือน เพื่อสร้างแบบจำลองทางคณิตศาสตร์ทั้งหมดก่อนส่งไปวาดกราฟ

7. แบบจำลองสภาพอากาศและรังสีแสงอาทิตย์ (Irradiance)

การประเมินพลังงานโซล่าร์เซลล์ที่แม่นยำ ต้องเริ่มต้นจากข้อมูลสภาพอากาศที่เชื่อถือได้ ในโค้ด V8.0 มีการเรียกใช้ PVGIS API /api/v5_2/seriescalc เพื่อดึงข้อมูลรายชั่วโมง (Hourly Data) มาทำการจำลอง

🌐 แหล่งข้อมูล API ที่ใช้งาน

  • PVGIS (EU JRC) Primary : API หลักที่ใช้ดึงค่า GHI (Global Horizontal Irradiance)
  • FallBack Mechanism: กรณี API เรียกไม่ได้ (CORS/Offline) ระบบจะนำค่า yearlyGHI_kWh_m2 จากไฟล์ DB2 (ค่าตั้งต้น 1,850 kWh/m²) มาเกลี่ยและจำลองแสงแบบ Sine Wave แทน

📐 แบบจำลองการแปลงรังสี (Transposition Model)

ระบบสกัดค่ารังสีในรูปแบบ DNI, DHI จากสมการความกดอากาศ (Airmass) และมุมดวงอาทิตย์ เพื่อหาความเข้มแสงตกกระทหน้าแผง (POA)

// สูตรการดึงค่าแสงตรงหน้าแผง (Beam) และ แสงกระจาย (Diffuse) จาก V8.0
let airmass = 1 / Math.cos(zenith);
DNI = 1367 * Math.pow(0.7, Math.pow(airmass, 0.678));
DHI = 0.3 * (1367 - DNI) * Math.sin(altitude);
GHI = DNI * Math.sin(altitude) + DHI;

// หักลบมุมตกกระทบแผง (POA)
poaBeam = DNI * cosAOI;
poaDiff = DHI * (1 + Math.cos(pTilt)) / 2;
poaRef = GHI * 0.2 * (1 - Math.cos(pTilt)) / 2;
POA_Total = poaBeam + poaDiff + poaRef;

8. แบบจำลองอุณหภูมิเซลล์ (Thermal Model)

อุณหภูมิมีผลผกผันกับประสิทธิภาพของแผงโซล่าร์เซลล์ โปรแกรมจึงเลือกใช้แบบจำลองของ Sandia National Laboratories ที่มีความแม่นยำสูง

🔥 King Sandia Thermal Model (Code Implementation)

ตามโค้ด V8.0 การสูญเสียจากอุณหภูมิจะถูกคำนวณในทุกชั่วโมง (Loop 8760) โดยอ้างอิงสัมประสิทธิ์ความร้อน `tc_pmax` จาก DB2

// ดึงค่า Tamb (อุณหภูมิอากาศ) แยกรายเดือน
let Tamb_dist = [26, 27, 29, 31, 30, 29, 28, 28, 27, 27, 27, 26];
let Tamb = Tamb_dist[m];

// คำนวณ T_cell ตามสูตร Sandia (e^(-3.56 + -0.075 * 1.5))
let tCell = Tamb + (poaTotal * Math.exp(-3.56 + -0.075 * 1.5));

// หักลบประสิทธิภาพแผงเมื่ออุณหภูมิเกิน STC (25°C)
let tempLoss = (tCell > 25) ? (tCell - 25) * Math.abs(tc_pmax/100) : 0;
let dc = polyCap * (poaTotal/1000) * (1 - tempLoss);

9. แบบจำลองความสูญเสียในระบบ (System Loss Model)

การแสดงผลแบบ Cascade/Waterfall Diagram เป็นมาตรฐานสากลอิงตามแบบจำลองของ PVSYST ช่วยให้วิศวกรเห็นจุดคอขวด (Bottleneck) ของพลังงานที่หายไปได้อย่างรวดเร็ว

🔻 องค์ประกอบความสูญเสียหลัก

  • Soiling Loss : ความสูญเสียจากฝุ่นละอองเกาะหน้าแผง (ตั้งค่าเริ่มต้นอิงตามประเทศไทยที่ 3-5%)
  • LID (Light Induced Degradation) : การเสื่อมสภาพในปีแรกเมื่อแผงสัมผัสกับแสงอาทิตย์ (ประมาณ 1-2%)
  • Wiring Ohmic Loss : ความต้านทานและการสูญเสียในสายไฟเคเบิล DC/AC
  • Inverter Loss : หักลบจากค่าประสิทธิภาพเฉลี่ยของ Inverter (เช่น 100% - 98.5% = 1.5% Loss)

ผลรวมของการหักลบทั้งหมดจะสะท้อนออกมาเป็นค่า Performance Ratio (PR) สุทธิของระบบ

10. แบบจำลองการจัดการแบตเตอรี่ (Battery Dispatch Algorithm)

โปรแกรมใช้สถาปัตยกรรม Rule-based Heuristic Dispatch จำลองการทำงานแบบรายชั่วโมง ซึ่งอ้างอิงจากโค้ด V8.0 ฟังก์ชัน `updateChartjsDailyInteractive()`

⚡ ตรรกะสมการรายชั่วโมง (Hourly Logic)

ตัวระบบจะควบคุมการคลายประจุ (Discharge) ไม่ให้ต่ำกว่า 25% (Min SoC) เพื่อยืดอายุรอบไซเคิล (Cycle Life) ของแบตเตอรี่ และแบ่งเวลาการคลายประจุตาม Time of Use (TOU)

// กำหนดความจุขั้นต่ำ (25%) และตรวจสอบเงื่อนไขเวลาคลายประจุ
let minSoC = totalBatCap * 0.25;
let canDischarge = (h < 9 && canDisBefore) || (h >= 15 && canDisAfter);

// พลังงานเหลือ (PV > Load) : นำไป Charge
if(d.pv > d.load) {
  let surplus = d.pv - d.load;
  let chargePwr = Math.min(surplus, invKw);
  if(currentSoC + chargePwr <= totalBatCap) currentSoC += chargePwr;
}
// พลังงานขาด (PV < Load) : นำมา Discharge
else if (d.pv < d.load && canDischarge) {
  let deficit = d.load - d.pv;
  let disPwr = Math.min(deficit, invKw);
  if(currentSoC - disPwr >= minSoC) currentSoC -= disPwr;
}

11. แบบจำลองทางเศรษฐศาสตร์ (DCF Financial Models)

ระบบใช้แบบจำลอง Discounted Cash Flow (DCF) เพื่อหาระยะเวลาคืนทุน (Payback Period) และกำไรสะสมตลอดอายุ 25 ปี อิงจากฟังก์ชัน `runSimulationEngine()` ใน V8.0

💰 สมการ DCF และปัจจัยเชิงซ้อน

// คำนวณ Degradation ของแผง (ปีที่ 1 เสื่อมเยอะกว่า ปีต่อไปเสื่อมคงที่)
if(y===1) { currentDegRate = 1 - (deg1/100); }
else { currentDegRate -= (deg2/100); }

// คำนวณอัตราเงินเฟ้อค่าไฟ (Tariff Inflation)
let currentTariff = tariff * Math.pow(1 + (tariffInf/100), y-1);
let savings = (totalEGrid * currentDegRate) * currentTariff;

// คำนวณ Net Cash Flow และหาจุดตัด Payback
let omCost = (pnom * 1000 * om) * Math.pow(1 + (infOM/100), y-1);
let netFlow = savings - omCost;
let dcf = netFlow / Math.pow(1 + (dr/100), y);
cumulative += dcf;

14. ขีดจำกัดความปลอดภัยของระบบ (System Voltage Limits)

อ้างอิงมาตรฐาน วสท. ส่วนที่ 3 ประเด็นด้านความปลอดภัย เพื่อป้องกันการเกิดประกายไฟฟ้าอาร์ก (Arc Flash) และการเกิดเพลิงไหม้ ระบบสารสนเทศจะต้องตรวจสอบและจำกัดแรงดันไฟฟ้ากระแสตรงสูงสุด (Max DC Voltage) ไม่ให้เกินพิกัดตามประเภทของอาคารอย่างเด็ดขาด

ตรรกะการคำนวณ (Logic Condition)

  • อาคารพักอาศัย (Residential): Vsys_limit = 1,000 Vdc
  • อาคารธุรกิจ/โรงงาน (Commercial/Industrial): Vsys_limit = 1,500 Vdc
// Pseudocode จาก V10.0 ฟังก์ชัน calcInvString()
let sysMaxVoltage = (buildingType === '1000') ? 1000 : 1500;
let effective_Max_DC = Math.min(Inverter_Max_DC_Input, sysMaxVoltage);

การประยุกต์ใช้: นักพัฒนาต้องนำค่า effective_Max_DC ไปใช้เป็นตัวตั้งในการคำนวณจำนวนแผงสูงสุดในทุกสมการถัดไป ห้ามใช้สเปกของ Inverter เดี่ยวๆ

15. แบบจำลองอุณหภูมิกับแรงดันแผง (Temp-Voltage Sizing)

อ้างอิงมาตรฐาน วสท. ส่วนที่ 4.2 แรงดันของแผงโซลาร์เซลล์จะแปรผกผันกับอุณหภูมิ การคำนวณจำนวนแผงที่เหมาะสมต้องใช้สมการ การชดเชยอุณหภูมิ (Temperature Compensation) เสมอ

15.1 การหาแรงดันสูงสุดในอุณหภูมิต่ำ (Max Array Voltage at Coldest Temp)

เพื่อป้องกันแรงดันทะลุเพดาน Inverter ในช่วงที่อากาศเย็นจัด

// สูตรจาก V10.0 (T_min_cold = 15°C, T_stc = 25°C)
let vocCold = pvVoc * (1 + (pvTempVoc / 100) * (15 - 25));

15.2 การหาแรงดันทำงานต่ำสุดในอุณหภูมิสูง (Min Array Voltage at Hottest Temp)

เพื่อป้องกันแรงดันตกลงจนต่ำกว่าจุดเริ่มทำงานของ Inverter (Min MPPT)

// สูตรจาก V10.0 (T_max_hot = 70°C, T_stc = 25°C)
let vmpHot = pvVmp * (1 + (pvTempVoc / 100) * (70 - 25));

16. ตรรกะการจัดสตริง (String Sizing Logic)

การหาขอบเขตจำนวนแผง (Range: Min-Max) ต่อ 1 สตริง แบ่งตามลักษณะฮาร์ดแวร์ที่เลือกใช้

16.1 กรณีระบบปกติ (Traditional String Inverter)

// จำนวนแผงสูงสุด (ป้องกัน Overvoltage ใช้ฟังก์ชันปัดเศษลง)
let maxStr = Math.floor( effectiveMaxDC / vocCold );

// จำนวนแผงต่ำสุด (ป้องกัน Inverter ดับ ใช้ฟังก์ชันปัดเศษขึ้น)
let minStr = Math.ceil( invMinMPPT / vmpHot );

16.2 กรณีใช้ Optimizer (Long String Design)

วงจร Buck/Boost ในตัว Optimizer จะทำหน้าที่ล็อคแรงดันออก (Voltage Clamping) ไม่ให้เกินสเปก ทำให้ลอจิกการคำนวณอุณหภูมิถูกยกเลิก

// โค้ดจาก V10.0 เมื่อเลือกใช้งาน Optimizer
let optMaxOutputVoltage = 80;
let maxPowerPerStringLimit = 10000;
let powerLimitedMaxStr = Math.floor(maxPowerPerStringLimit / pvPmax);

// กรณี Ratio 1:1
let voltLimitedMax = Math.floor(effectiveMaxDC / optMaxOutputVoltage);
maxStr = Math.min(voltLimitedMax, powerLimitedMaxStr);

// DC Breaker (เผื่อ 1.5 เท่าของกระแสลัดวงจร) และปัดขึ้นทีละ 5

18. อัตราส่วนกำลังไฟฟ้า (DC/AC Sizing Ratio)

ดัชนีชี้วัดความคุ้มค่าทางเศรษฐศาสตร์และการใช้งานประสิทธิภาพ Inverter (ป้องกัน Clipping Loss ที่รุนแรง)

Ratio = (Total_Panels * P_MOD_STC_kW) / Total_Inverter_AC_kW;

เงื่อนไขการแจ้งเตือน (Status Triggers)

  • Ratio < 1.20 : ⚠️ ต่ำกว่าเกณฑ์ - Inverter ใหญ่เกินไป สิ้นเปลืองงบประมาณ ควรเพิ่มแผง
  • 1.20 ≤ Ratio ≤ 1.40 : ✅ ปกติ (Optimal) - เป็นช่วงที่ วสท. และผู้ผลิตแนะนำว่าคุ้มค่าที่สุด
  • Ratio > 1.40 : 🔴 สูงกว่าเกณฑ์ - แผงเยอะเกินไป Inverter จะตัดการทำงานทิ้งในช่วงเที่ยง (Clipping) เสียพลังงานทิ้งฟรี

19. การคำนวณขนาดท่อร้อยสายและรางเคเบิล (Conduit & Wireway Sizing)

อ้างอิงมาตรฐาน วสท. หมวดที่ 5 การร้อยสายไฟ (PV Cable) ลงในท่อหรือรางจะต้องมีพื้นที่ว่าง (Free Space) เพื่อระบายอากาศตามกฎเกณฑ์ดังต่อไปนี้

19.1 พื้นที่หน้าตัดรวม (Total Wire Area)

// ตัวอย่างจากโค้ด V10.0 : หาพื้นที่หน้าตัดสายรวมฉนวน
// สาย 4 sq.mm (OD~23), สาย 6 sq.mm (OD~28)
let dcCableOdArea = (dcCableSize === 4) ? 23 : 28;

let totalDcWireArea = (numStrings * 2 * dcCableOdArea) + gndCableOdArea;

19.2 กฎ 40% (Conduit) และ 20% (Wireway)

// 1. กรณีท่อร้อยสาย (Conduit) ใช้เกณฑ์ 40%

20. เรขาคณิตของแสงอาทิตย์และมุมตกกระทบ (Solar Geometry & Angle of Incidence;AOI)

เอนจินของโปรแกรมไม่ได้ทำงานแค่บนแผนที่ราบแบบ 2D แต่มีการดึงไลบรารี suncalc.js มาเพื่อหาตำแหน่งดวงอาทิตย์สัมพัทธ์กับที่ตั้งโครงการในทุกชั่วโมง

20.1 การหาพิกัดดวงอาทิตย์ (Sun Position)

// ดึงข้อมูล Zenith และ Azimuth ของดวงอาทิตย์ (V8.0)
let sunPos = SunCalc.getPosition(date, lat, lon);
let zenith = (Math.PI/2) - sunPos.altitude;
let sunAzi = sunPos.azimuth + Math.PI; // ปรับเทียบทิศเหนือ

20.2 การหามุมตกกระทบหน้าแผง (Angle of Incidence: AOI)

ระบบนำทิศทางของดวงอาทิตย์มาหักลบกับทิศการเอียงของหลังคา (Roof Slope / Azimuth) ด้วยคณิตศาสตร์ตรีโกณมิติเชิงทรงกลม

// pTilt = ความเอียงหลังคา, pAzi = ทิศหันหลังคา
let cosAOI = Math.cos(zenith)*Math.cos(pTilt) + Math.sin(zenith)*Math.sin(pTilt)*Math.cos(sunAzi - pAzi);
if (cosAOI < 0) cosAOI = 0;
let aoiDeg = Math.acos(cosAOI) * 180 / Math.PI;

// แปลงมุมที่ตกกระทบเบี้ยวให้กลายเป็นเปอร์เซ็นต์ Loss (AOI Efficiency)
let aoiEff = Math.max(0, 100 - (aoiDeg/90)*100);

21. อัลกอริทึมวาดแผงบนแผนที่ (Web-GIS Grid Layout)

Auto Solar Copilot ใช้ไลบรารี Turf.js เข้ามาจัดการพื้นที่ Polygon บนแผนที่ Leaflet เพื่อเปลี่ยนหลังคาที่ผู้ใช้วาดให้เป็นตารางเรียงแผงแบบสลับฟันปลา (Snake Pattern) อัตโนมัติ

21.1 การสร้างระยะเว้นขอบ (Keepout & Buffer Offsets)

// หดขอบหลังคาเข้ามาด้านใน (Inner Edge Gap) ด้วยค่าติดลบ
const bufferedRoof = turf.buffer(roofGeojson, -innerEdgeGapM, {units: 'meters'});

// ขยายพื้นที่กีดขวาง (Obstacles / Walkway) ดันแผงออกด้วยค่าบวก
const expandedKeepout = turf.buffer(ko, keepoutGapM, {units: 'meters'});

21.2 การสร้างตารางแผง (Grid Generation Loop)

ระบบจะสร้างกล่อง Bounding Box คลุมพื้นที่ทั้งหมด แล้วเดินลูปเช็คทีละแผงว่าจุดกึ่งกลางของแผง (Centroid) ตกอยู่ในขอบเขตหลังคาที่ยอมรับได้หรือไม่ (Point-in-Polygon)

// Loop แกน X (คอลัมน์) และ แกน Y (แถว)
while (currentX < bbox[2]) {
  while (currentY < bbox[3]) {
    let checkPoly = turf.polygon([...]);
    if (turf.booleanPointInPolygon(turf.center(checkPoly), rotatedRoof)) {
      colPanels.push(realWorldCheckPanel);
    }
    currentY += stepY; // ขยับแนวตั้ง (บวก Walkway ถ้ายาวครบกำหนด)
  }
  if (colIndex % 2 !== 0) colPanels.reverse(); // สลับ Snake Pattern
  panels = panels.concat(colPanels);
  currentX += stepX; // ขยับแนวนอน
}

22. มาตรฐานการคุมแรงดันตก (Cable & Voltage Drop Rules)

เพื่อป้องกันพลังงานสูญเสียเป็นความร้อนในสายเคเบิล ระบบจะคำนวณย้อนกลับ (Reverse Calculation) จากค่า Voltage Drop ที่ยอมรับได้ เพื่อหาความยาวสายสูงสุดที่ลากได้ หรือบังคับอัปไซส์สาย

22.1 กฎ DC Voltage Drop (ไม่เกิน 1%)

// สมการหาสายไฟ DC จาก V10.0 (ใช้สายทองแดง ความต้านทานจำเพาะ 0.0216)
// Limit_DC_Length = (Area * V_Drop_Limit * Vmp) / (2 * Imp * 0.0216)
let maxDcLengthAllowed = Math.floor((6 * 0.01 * stringVmp) / (2 * stringImp * 0.0216));

if (dcLength > maxDcLengthAllowed) {
  showWarning("สาย DC ยาวเกินไป บังคับลดระยะเพื่อคุม V-Drop 1%");
  dcLength = maxDcLengthAllowed;
}

22.2 กฎ AC Voltage Drop (ไม่เกิน 3%)

// สมการหาสายไฟ AC (acFactor: 1 Phase=2, 3 Phase=1.732)
let maxAcLengthAllowed = Math.floor((16 * (0.03 * acVolts)) / (acFactor * maxAcCurrent * 0.0216));

23. โครงสร้างจับยึด & ความปลอดภัยหลังคา (Mounting & Access BOM)

ข้อมูลชิ้นส่วนโครงสร้างและระบบความปลอดภัยในการขึ้นลงหลังคา (Roof Access Safety) เป็นส่วนสำคัญของ BOQ ซึ่งโปรแกรมสามารถประเมินได้จากตรรกะต่อไปนี้

23.1 การคำนวณโครงสร้างจับยึด (Mounting Hardware)

// คำนวณจำนวนราง (Rail) จากความกว้างแผงรวมและช่องไฟ
let railQty = Math.ceil(totalRailLengthM / stdRailLength);

// คำนวณข้อต่อราง (Rail Splice) เผื่อเศษ 20% หากรอยตัดเหลือน้อย
let railSpliceQty = railQty - (totalSegments * 2);
if (railSpliceQty <= 0) railSpliceQty = Math.ceil(railQty * 0.20);

// ขายึด (L-Feet / Roof Hook) ตามระยะ Span (เช่น 1.2 เมตร)
let lFeetQty = Math.ceil(totalRailLengthM / spanDist);

23.2 มาตรฐานความปลอดภัยการขึ้นลงหลังคา (Roof Access & Safety Standards)

อ้างอิงตามมาตรฐาน OSHA และกฎกระทรวงความปลอดภัยฯ ของไทย หากหลังคามีความสูง (Roof Height) แตกต่างจากพื้น จะต้องประเมินบันไดและอุปกรณ์ป้องกันตกลงใน BOM ทันที

// 1. บันไดเซอร์วิสแนวดิ่ง (Vertical Fixed Ladder)
if (Roof_Height > 3) {
  Ladder_Length_m = Roof_Height + 1.0; // เผื่อราวจับเหนือหลังคา 1 เมตร
}

// 2. กรงป้องกันตก (Safety Cage)
// บังคับติดกรงหากบันไดสูงเกิน 6 เมตร โดยกรงจะเริ่มจากระยะ 2.5m จากพื้น
if (Roof_Height > 6) {
  Cage_Length_m = Roof_Height - 2.5;
}

// 3. ชานพัก (Landing Platform)
// หากต้องปีนต่อเนื่องยาวเกิน 6 เมตร ต้องมีชานพักพักเหนื่อย
if (Roof_Height > 6) {
  Landing_Platforms_Qty = Math.floor(Roof_Height / 6);
}

24. ระบบล้างแผงอัตโนมัติ (Cleaning System Sizing)

ระบบท่อน้ำสำหรับฉีดล้างหน้าแผงแบบเดินท่อถาวร (Sprinkler) จะถูกประเมินขนาดปั๊มและระยะท่อโดยอ้างอิงความยาวรางแผงโซลาร์เซลล์

24.1 การหาระยะท่อและก๊อกน้ำล

// 1. ก๊อกน้ำล้างแผง (Water Tap): เกรดอุตสาหกรรม (ทุก 10 เมตร)
tapQty = Math.ceil(totalRailLengthM / 10);

// 2. ท่อไมโคร PE บนหลังคา: เดินขนานตามแนวราง เผื่อเลี้ยว 10%
let pePipeLength = Math.ceil(totalRailLengthM * 1.1);

// 3. ท่อเมนน้ำ UPVC: วิ่งแนวดิ่งลงปั๊ม เผื่อ 20%
let mainPipeLength = Math.ceil(pipeDistDown * 1.2);

24.2 การเลือกขนาดปั๊มน้ำแรงดันสูง (Pump Sizing)

เอนจินจะประเมินพิกัดปั๊มตามปริมาณแผงที่ติดตั้ง เพื่อรักษาแรงดันหัวฉีดปลายสายไม่ให้ตก

let pumpSpec = "1.0 HP (750W)";
if (totalPanels > 100) pumpSpec = "5.0 HP (3-Phase)";
else if (totalPanels > 50) pumpSpec = "3.0 HP (2200W)";
else if (totalPanels > 20) pumpSpec = "2.0 HP (1500W)";

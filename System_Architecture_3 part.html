1. สถาปัตยกรรมระบบ (System Overview)

Auto Solar Copilot เป็นเครื่องมือซอฟต์แวร์วิศวกรรมบนเว็บ (Web-based Engineering Tool) ที่ถูกออกแบบมาเพื่อการออกแบบระบบผลิตไฟฟ้าจากพลังงานแสงอาทิตย์บนหลังคา (Solar Rooftop) และระบบติดตั้งบนพื้นดิน

ระบบไม่ได้เป็นเพียงเครื่องมือวาดแผนที่ (Drawing Tool) แต่ผสานการทำงานของระบบสารสนเทศภูมิศาสตร์ (GIS) ผ่านไลบรารี Leaflet เข้ากับเอนจินคณิตศาสตร์วิศวกรรมไฟฟ้าแบบเรียลไทม์ ทำให้ผู้ใช้สามารถประเมินพื้นที่จริง วาดแผงโซลาร์เซลล์ และตรวจสอบความปลอดภัยทางไฟฟ้าได้ในหน้าจอเดียว

จุดเด่นของสถาปัตยกรรม (Architectural Highlights)

  • Hybrid Web-GIS: ใช้ Leaflet (JavaScript Library แบบ Open-source สำหรับการพัฒนาแผนที่แบบโต้ตอบ (Interactive Web Map)) นำพิกัดตำแหน่งของอาคาร,พื้นที่ มาวาดภาพจำลอง (Polygon Rendering) และจัดการด้วย SVG/Canvas เพื่อให้แผนที่คงสัดส่วนเดิม
  • EIT Sizing Engine: ฝัง (รหัสเทียม (Pseudocode))มาตรฐานวิศวกรรมสถานแห่งประเทศไทย (วสท.) ไว้ในระบบคณิตศาสตร์ของโปรแกรม เพื่อควบคุมแรงดันกระแสตรงไม่ให้เกินขีดจำกัด
  • Zero-Loss Data Binding: โครงสร้าง Database รูปแบบ JSON (DB1 และ DB2) ที่ทำให้สามารถทำเข้า-ส่งออก (Import/Export) โปรเจกต์ได้ 100% โดยที่พารามิเตอร์ไม่สูญหาย วางแผนเมื่อต้องนำระบบไปพัฒนาต่อในรูปแบบ Client-Server

2. แบบจำลองด้านพลังงาน (Energy Models & Equations)

การคำนวณในโปรแกรมนี้ถูกออกแบบโดยอ้างอิงมาตรฐานวิศวกรรมสากล (IEC 62548) และมาตรฐาน วสท. ของประเทศไทย เพื่อป้องกันความเสียหายแก่อินเวอร์เตอร์และลดความเสี่ยงการเกิดเพลิงไหม้

2.1 แบบจำลองการจัดสตริง (String Sizing Model)

การหาจำนวนแผงสูงสุดและต่ำสุดต่อสตริง โปรแกรมไม่ได้ใช้แรงดันสภาวะ STC (25°C) มาหารตรงๆ แต่ใช้ ค่าสัมประสิทธิ์อุณหภูมิ (Temperature Coefficient) ในการจำลองแรงดันที่เปลี่ยนแปลงไปตามสภาพอากาศของประเทศไทย:

// สมการหาจำนวนแผงสูงสุด (หน้าหนาว T_min = 15°C)

VOC_cold = VOC_STC × [ 1 + ( γv / 100 ) × (Tmin - 25) ]

Max_Panels = Floor( Vmax_system / VOC_cold )
// สมการหาจำนวนแผงต่ำสุด (หน้าร้อน T_max = 70°C)

VMP_hot = VMP_STC × [ 1 + ( γv / 100 ) × (Tmax - 25) ]

Min_Panels = Ceil( Vmin_MPPT / VMP_hot )

2.2 แบบจำลอง Optimizer Voltage Clamping

เมื่อระบบตรวจพบการใช้งาน Power Optimizer โปรแกรมจะข้ามการประเมินอุณหภูมิแผง และเปลี่ยนไปใช้เงื่อนไขคุมแรงดันและกำลังไฟฟ้าของ Optimizer (Voltage Clamping) ทำให้สามารถต่อสตริงได้ยาวขึ้น

Max_Panels = Min( Floor(Vmax_INV / Vmax_OPT_output) , Floor(Pmax_StringLimit / Pmax_PV) )

2.3 แบบจำลองอุณหภูมิแผงโซลาร์เซลล์ (Sandia Model โดย David L. King)

โปรแกรมใช้โมเดลของห้องปฏิบัติการ Sandia National Laboratories เพื่อประเมินอุณหภูมิแผงจริง (Tcell) ซึ่งคำนึงถึงความเข้มแสงและความเร็วลม ทำให้ประเมินการสูญเสียในสภาวะการทำงานปกติได้แม่นยำกว่า

Tcell = Tamb + GHI × e(a + b × WS)

แบบจำลองประสิทธิภาพไฟฟ้า (Sandia Array Performance Model: SAPM): คำนวณหาค่ากระแสไฟ (Short-circuit current) และแรงดันไฟฟ้า (Open-circuit voltage) (Tcell) คำนวณหาค่ากระแสไฟ (Short-circuit current) และแรงดันไฟฟ้า (Open-circuit voltage) ออกมาเป็นจุดทำงานต่าง ๆ บนกราฟ I-V Curve เพื่อหาพลังงานไฟฟ้ากระแสตรง (DC Power) ที่แผงผลิตได้ * กำลังพัฒนา

3. ข้อมูลสภาพแวดล้อมและ API (Meteo Data APIs)

ข้อมูลสภาพแวดล้อม (Meteo Data) เป็นหัวใจสำคัญในการจำลองการผลิตไฟฟ้า (Energy Yield) โปรแกรมออกแบบสถาปัตยกรรมให้รองรับ API 3 แหล่ง ซึ่งเป็นมาตรฐานอุตสาหกรรมสากล:

แหล่งข้อมูล API ความละเอียด เหตุผลในการเลือกนำมาใช้งาน
PVGIS (JRC Europe) รายชั่วโมง (TMY) เป็น API ที่น่าเชื่อถือระดับโลก ใช้งานได้ฟรี 100% ไม่มี API Key ถูกตั้งเป็น Default Engine
NASA POWER รายชั่วโมง (Satellite) ครอบคลุมพื้นที่ทั่วโลกแบบ 100% แม้อยู่ในพื้นที่ห่างไกลหรือกลางทะเล
Meteonorm รายชั่วโมง (Interpolation) ถือเป็นมาตรฐานทองคำ (Gold Standard) ในการออกเอกสารกู้เงินธนาคาร

ชื่อโปรแกรม ประเภทสิทธิการใช้ การวิเคราะห์และเปรียบเทียบ

5. แบบจำลองทางการเงิน (Financial & ROI Model)

ในการอัปเกรดจากเครื่องมือทางวิศวกรรมสู่ "เครื่องมือประเมินการลงทุน" ระบบสถาปัตยกรรมจะถูกขยายให้รองรับการคำนวณด้านกระแสเงินสด (Cash Flow) โดยมีสมการชี้วัดทางการเงิน ดังนี้:

5.1 โครงสร้างต้นทุนและรายได้

  • CAPEX (ต้นทุนเริ่มต้น): คำนวณจาก Total DC Capacity (kWp) × ราคาต่อวัตต์
  • OPEX (ต้นทุนดำเนินงาน): ครอบคลุมค่า O&M (เช่น 1% ของ CAPEX/ปี) และค่า Inverter Replacement ในปีที่ 10
  • Revenue (รายได้/ส่วนประหยัด): แบ่งสัดส่วนระหว่าง Self-Consumption (ทดแทนการซื้อไฟ) และ Grid Export (ขายไฟ)

5.2 ดัชนีชี้วัดเพื่อการลงทุน (Financial Metrics)

1. มูลค่าปัจจุบันสุทธิ (Net Present Value : NPV)

เมทริกซ์หลักที่ใช้บอกว่าโครงการคุ้มค่าการลงทุนหรือไม่ในช่วงระยะเวลา 25 ปี

NPV = Σ [ (Rt - Ct) / (1 + d)t ] - CAPEX

2. อัตราผลตอบแทนภายใน (Internal Rate of Return : IRR)

ใช้อัลกอริทึม (เช่น Newton-Raphson) ในการค้นหาเปอร์เซ็นต์ IRR ที่ทำให้ NPV มีค่าเท่ากับ 0

3. ต้นทุนการผลิตไฟฟ้าต่อหน่วย (Levelized Cost of Energy : LCOE)

ใช้เปรียบเทียบว่าต้นทุนการสร้างโรงไฟฟ้าตกอยู่ที่กี่บาทต่อ 1 หน่วยไฟฟ้า (บาท/kWh)

LCOE = [ CAPEX + Σ (OPEXt / (1 + d)t) ] / Σ (Et / (1 + d)t)

6. โครงสร้างสถาปัตยกรรมระบบ (Direct-to-Memory)

Auto Solar Copilot ถูกพัฒนาด้วยแนวคิด "Direct-to-Memory (Client-side Simulation)" ซึ่งประมวลผลทุกอย่างบนเบราว์เซอร์ของผู้ใช้ 100% โดยไม่ต้องพึ่งพาเซิร์ฟเวอร์ (Zero Backend Cost) ส่งผลให้มีความปลอดภัยของข้อมูลสูง (Data Privacy) และประมวลผลได้รวดเร็ว

💡 กระบวนการทำงานหลัก (Data Pipeline)

  • Data Ingestion: นำเข้าไฟล์พารามิเตอร์ (.txt/.json) ผ่าน FileReader เข้าสู่หน่วยความจำ (RAM) โดยตรง
  • Regex Sweeper: ใช้ Regular Expression ระดับสูงในการตัดขยะ (เช่น mapGeoJSON ที่ทำให้เครื่องค้าง) และสกัดเฉพาะพารามิเตอร์ของอุปกรณ์
  • DB3 Memory State: จัดเก็บข้อมูลลงโครงสร้างอ็อบเจกต์ใน sessionStorage ทำให้สามารถสลับหน้าจอ (UI Tabs) ไปมาได้โดยข้อมูลไม่สูญหาย
  • Simulation Engine: วิ่งลูปคำนวณ 8,760 ชั่วโมง (Hourly Step) และระดับเดือน เพื่อสร้างแบบจำลองทางคณิตศาสตร์ทั้งหมดก่อนส่งไปวาดกราฟ

7. แบบจำลองสภาพอากาศและรังสีแสงอาทิตย์ (Irradiance)

การประเมินพลังงานโซล่าร์เซลล์ที่แม่นยำ ต้องเริ่มต้นจากข้อมูลสภาพอากาศที่เชื่อถือได้ ระบบจึงออกแบบให้มีการเชื่อมต่อ API อัตโนมัติหากไฟล์ต้นฉบับไม่มีข้อมูล

🌐 แหล่งข้อมูล API ที่ใช้งาน

  • PVGIS (EU JRC) Primary : API หลักที่ใช้ดึงค่า GHI (Global Horizontal Irradiance) มีความแม่นยำสูงมากในระดับสากล
  • NASA POWER Fallback : ฐานข้อมูลจากดาวเทียม NASA ทั่วโลก ใช้เป็นแผนสำรองกรณีที่ PVGIS ไม่ตอบสนอง

📐 แบบจำลองการแปลงรังสี (Transposition Model)

ระบบใช้แนวคิดอ้างอิงจาก Perez Model (Isotropic Sky Model) เพื่อแปลงค่ารังสีรวมแนวราบ (GHI) ให้เป็นรังสีตกกระทบบนระนาบเอียง (Global Incident - GlobInc)

GlobInc = GHI × [1 + (Tilt × 0.002)]

*หมายเหตุ: สมการถูกปรับให้อยู่ในรูป Linear Approximation สำหรับภูมิภาคเขตศูนย์สูตร เพื่อลดภาระการคำนวณบนเบราว์เซอร์

8. แบบจำลองอุณหภูมิเซลล์ (Thermal Model)

อุณหภูมิมีผลผกผันกับประสิทธิภาพของแผงโซล่าร์เซลล์ (ยิ่งร้อน ยิ่งผลิตไฟได้น้อยลง) โปรแกรมจึงเลือกใช้แบบจำลองของ Sandia National Laboratories ที่มีความแม่นยำสูงกว่าแบบจำลอง NOCT ธรรมดา

🔥 King Sandia Thermal Model

แบบจำลองนี้มีการนำความเร็วลม (Wind Speed) และอุณหภูมิแวดล้อม (Tamb) แยกรายเดือนมาวิเคราะห์ ทำให้กราฟ PR สวิงขึ้นลงตามฤดูกาลอย่างสมจริง

Tm = Tamb + Einc × e(a + b × WS)

Tcell = Tm + (Einc / E0) × ΔT
  • Tamb : อุณหภูมิอากาศ (รายเดือน)
  • Einc : รังสีตกกระทบหน้าแผง (W/m²)
  • WS : ความเร็วลม (m/s)
  • a, b, ΔT : ค่าสัมประสิทธิ์เชิงโครงสร้างการติดตั้งแผง (Empirical Parameters)

📉 การคำนวณการลดทอน (Degradation)

Thermal_Loss_Pct = Max(0, (Tcell - 25°C) × |Temp_Coeff_Pmax|)

9. แบบจำลองความสูญเสียในระบบ (System Loss Model)

การแสดงผลแบบ Cascade/Waterfall Diagram เป็นมาตรฐานสากลอิงตามแบบจำลองของ PVSYST ช่วยให้วิศวกรเห็นจุดคอขวด (Bottleneck) ของพลังงานที่หายไปได้อย่างรวดเร็ว

🔻 องค์ประกอบความสูญเสียหลัก

  • Soiling Loss : ความสูญเสียจากฝุ่นละอองเกาะหน้าแผง (ตั้งค่าเริ่มต้นอิงตามประเทศไทยที่ 3-5%)
  • LID (Light Induced Degradation) : การเสื่อมสภาพในปีแรกเมื่อแผงสัมผัสกับแสงอาทิตย์ (ประมาณ 1-2%)
  • Wiring Ohmic Loss : ความต้านทานและการสูญเสียในสายไฟเคเบิล DC/AC
  • Inverter Loss : หักลบจากค่าประสิทธิภาพเฉลี่ยของ Inverter (เช่น 100% - 98.5% = 1.5% Loss)

ผลรวมของการหักลบทั้งหมดจะสะท้อนออกมาเป็นค่า Performance Ratio (PR) สุทธิของระบบ

10. แบบจำลองการจัดการแบตเตอรี่ (Battery Dispatch Algorithm)

โปรแกรมใช้สถาปัตยกรรม Rule-based Heuristic Dispatch จำลองการทำงานแบบรายชั่วโมง (Hourly Step) ซึ่งเป็นวิธีที่สะท้อนพฤติกรรมโหมด Self-Consumption หรือ TOU ใน Inverter Hybrid ได้แม่นยำ

⚡ ตรรกะสมการรายชั่วโมง (Hourly Logic)

Esurplus = EPV - ELoad
  • กรณี Esurplus > 0 (พลังงานเหลือ) :
    นำกระแสไฟฟ้าไปชาร์จแบตเตอรี่ (Charge) โดยจำกัดกระแสไม่ให้เกินพิกัด Max_Charge ของอินเวอร์เตอร์ และไม่ให้ความจุสะสมเกิน Total_Capacity
  • กรณี Esurplus < 0 (พลังงานขาด) :
    ดึงไฟจากแบตเตอรี่มาจ่ายโหลด (Discharge) เฉพาะใน "ช่วงเวลาที่อนุญาต (Allowed Windows)" ซึ่งสัมพันธ์กับ Checkbox บน UI เพื่อรองรับการทำ Peak Shaving

11. แบบจำลองทางเศรษฐศาสตร์ (DCF Financial Models)

ระบบใช้แบบจำลอง Discounted Cash Flow (DCF) ซึ่งเป็นมาตรฐานบัญชีสากลในการประเมินความคุ้มค่าโครงการ ที่สถาบันการเงินใช้ประกอบการพิจารณาสินเชื่อ

💰 สมการ DCF และปัจจัยเชิงซ้อน

NPV = Σ [ CashFlowt / (1 + r)t ] - CAPEX
  • Energy Degradation : การเสื่อมสภาพของแผงโซล่าร์รายปี
  • Tariff Inflation : อัตราการเพิ่มขึ้นของค่าไฟฟ้าการไฟฟ้ารายปี (Compound Interest)
  • O&M Inflation : อัตราเงินเฟ้อของค่าบำรุงรักษา (เช่น ค่าล้างแผง)
  • Discount Rate (r) : อัตราคิดลดเพื่อแปลงมูลค่าเงินในอนาคตกลับมาเป็นมูลค่าปัจจุบัน

12. แบบจำลองด้านสิ่งแวดล้อม & มาตรฐานอุตสาหกรรม

รายงานที่สร้างขึ้นรองรับการนำไปใช้ประกอบแผน ESG (Environmental, Social, and Governance) และ Carbon Neutrality ขององค์กรได้อย่างถูกต้อง

🌿 การประเมินคาร์บอน (Environmental Impact)

  • TGO (อบก.) : อ้างอิงค่า Grid Emission Factor ล่าสุดของประเทศไทยปี 2568-2569 0.4750 kgCO2/kWh
  • TGO (อบก.) : Scope 2 (การใช้ไฟฟ้า): 0.4750 kgCO₂eq/kWh (หรือ 0.4750 tCO₂eq/MWh) Scope 3 (การได้มาซึ่งพลังงาน): 0.0812 kgCO₂eq/kWh
  • T-VER : ประเมินมูลค่าผลพลอยได้จากคาร์บอนเครดิตที่ราคา 120 บาท/ตัน
  • US EPA : ใช้โมเดลสากลขององค์กรพิทักษ์สิ่งแวดล้อมสหรัฐฯ เทียบเท่าการปลูกต้นไม้ และการลดระยะทางรถยนต์สันดาป

📏 ความสอดคล้องมาตรฐาน วสท. 022013-21

  • DC/AC Ratio : สอดคล้องกับข้อกำหนดขีดจำกัดการรับกระแสลัดวงจรของ Inverter
  • Wiring Voltage Drop : อนุญาตให้ผู้ใช้ออกแบบค่าสูญเสียในสายไฟ (Ohmic Loss) ให้อยู่ในเกณฑ์ 3-5% ตามมาตรฐาน

13. การเปรียบเทียบเชิงเทคนิคกับซอฟต์แวร์ตลาด (Technical Benchmarking)

โปรแกรม (Software) จุดเด่น / ค่าใช้จ่าย เปรียบเทียบเชิงสถาปัตยกรรมกับAuto Solar Copilot

14. ขีดจำกัดความปลอดภัยของระบบ (System Voltage Limits)

อ้างอิงมาตรฐาน วสท. ส่วนที่ 3 ประเด็นด้านความปลอดภัย เพื่อป้องกันการเกิดประกายไฟฟ้าอาร์ก (Arc Flash) และการเกิดเพลิงไหม้ ระบบสารสนเทศจะต้องตรวจสอบและจำกัดแรงดันไฟฟ้ากระแสตรงสูงสุด (Max DC Voltage) ไม่ให้เกินพิกัดตามประเภทของอาคารอย่างเด็ดขาด

ตรรกะการคำนวณ (Logic Condition)

  • อาคารพักอาศัย (Residential): Vsys_limit = 1,000 Vdc
  • อาคารธุรกิจ/โรงงาน (Commercial/Industrial): Vsys_limit = 1,500 Vdc
// ตัวอย่าง Pseudocode สำหรับกำหนดเพดานแรงดัน
let V_sys_limit = (buildingType === 'residential') ? 1000 : 1500;
let effective_Max_DC = Math.min(Inverter_Max_DC_Input, V_sys_limit);

การประยุกต์ใช้: นักพัฒนาต้องนำค่า effective_Max_DC ไปใช้เป็นตัวตั้งในการคำนวณจำนวนแผงสูงสุดในทุกสมการถัดไป ห้ามใช้สเปกของ Inverter เดี่ยวๆ

15. แบบจำลองอุณหภูมิกับแรงดันแผง (Temp-Voltage Sizing)

อ้างอิงมาตรฐาน วสท. ส่วนที่ 4.2 แรงดันของแผงโซลาร์เซลล์จะแปรผกผันกับอุณหภูมิ (ยิ่งหนาวแรงดันยิ่งสูง ยิ่งร้อนแรงดันยิ่งตก) การคำนวณจำนวนแผงที่เหมาะสมต้องใช้สมการ การชดเชยอุณหภูมิ (Temperature Compensation) เสมอ

15.1 การหาแรงดันสูงสุดในหน้าหนาว (Max Array Voltage at Coldest Temp)

เพื่อป้องกันแรงดันทะลุเพดาน Inverter ในช่วงที่อากาศเย็นจัด

  • Tmin_cold = อุณหภูมิแวดล้อมต่ำสุดที่คาดการณ์ (แนะนำ 10°C หรือ 15°C สำหรับประเทศไทย)
  • γv = สัมประสิทธิ์อุณหภูมิของแรงดันเปิดวงจร (Temp Coeff Voc) *ค่าเป็นลบ
// สูตรคำนวณแรงดันเปิดวงจรในหน้าหนาว
V_OC_cold = V_OC_STC * (1 + (Temp_Coeff_Voc / 100) * (T_min_cold - 25));

15.2 การหาแรงดันทำงานต่ำสุดในหน้าร้อน (Min Array Voltage at Hottest Temp)

เพื่อป้องกันแรงดันตกลงจนต่ำกว่าจุดเริ่มทำงานของ Inverter (Min MPPT) ในช่วงเที่ยงวันของฤดูร้อน

  • Tmax_hot = อุณหภูมิเซลล์สูงสุดในหน้าร้อน (แนะนำ 70°C)
// สูตรคำนวณแรงดันทำงานสูงสุดในหน้าร้อน
V_MP_hot = V_MP_STC * (1 + (Temp_Coeff_Voc / 100) * (T_max_hot - 25));

16. ตรรกะการจัดสตริง (String Sizing Logic)

การหาขอบเขตจำนวนแผง (Range: Min-Max) ต่อ 1 สตริง แบ่งตามลักษณะฮาร์ดแวร์ที่เลือกใช้

16.1 กรณีระบบปกติ (Traditional String Inverter)

// จำนวนแผงสูงสุด (ป้องกัน Overvoltage ใช้ฟังก์ชันปัดเศษลง)
Max_Panels_Per_String = Math.floor( effective_Max_DC / V_OC_cold );

// จำนวนแผงต่ำสุด (ป้องกัน Inverter ดับ ใช้ฟังก์ชันปัดเศษขึ้น)
Min_Panels_Per_String = Math.ceil( Inverter_Min_MPPT / V_MP_hot );

16.2 กรณีใช้ Optimizer (Long String Design)

วงจร Buck/Boost ในตัว Optimizer จะทำหน้าที่ล็อคแรงดันออก (Voltage Clamping) ไม่ให้เกินสเปก ทำให้ลอจิกการคำนวณอุณหภูมิถูกยกเลิก และใช้ขีดจำกัดด้านกำลังไฟฟ้า (Power Limit) แทน

// V_OPT_max_out = แรงดันส่งออกสูงสุดของ Optimizer (เช่น 80V)
// P_INV_max_str = ขีดจำกัดกำลังไฟฟ้ารวมต่อ 1 สตริงของ Inverter (เช่น 10000W)

let volt_limited_max = Math.floor(effective_Max_DC / V_OPT_max_out);
let power_limited_max = Math.floor(P_INV_max_str / P_MOD_STC);

// เลือกค่าที่ทำให้แผงน้อยกว่าเพื่อความปลอดภัย แล้วคูณอัตราส่วน Optimizer (1 หรือ 2)
Max_Panels_Per_String = Math.min(volt_limited_max, power_limited_max) * Opt_Ratio;

17. การป้องกันกระแสเกิน (Overcurrent Protection & Fuses)

อ้างอิง วสท. ส่วนที่ 3.3.3 หากมีการนำสตริงมาต่อขนานกัน (Parallel) ในพอร์ต MPPT เดียวกันมากกว่า 2 สตริง อาจเกิดกระแสไหลย้อนกลับไปทำลายแผงโซลาร์เซลล์ได้

ตรรกะการตรวจสอบ (Validation Logic)

// S_A = จำนวนสตริงที่ต่อขนานเข้า MPPT เดียวกัน
// I_SC_MOD = กระแสลัดวงจรของแผง
// I_MOD_MAX_OCPR = พิกัดฟิวส์สูงสุดที่แผงทนได้

if ( (S_A - 1) * I_SC_MOD > I_MOD_MAX_OCPR ) {
    triggerWarning("ต้องติดตั้ง String Fuses ตามมาตรฐาน วสท.");
}

ข้อแนะนำสำหรับผู้พัฒนา: หากตัวแปร Inputs_per_MPPT ของ Inverter ที่เลือกมีค่า > 2 ให้ระบบแจ้งเตือนบังคับใส่ฟิวส์ทันที

18. อัตราส่วนกำลังไฟฟ้า (DC/AC Sizing Ratio)

ดัชนีชี้วัดความคุ้มค่าทางเศรษฐศาสตร์และการใช้งานประสิทธิภาพ Inverter (ป้องกัน Clipping Loss ที่รุนแรง)

Ratio = (Total_Panels * P_MOD_STC_kW) / Total_Inverter_AC_kW;

เงื่อนไขการแจ้งเตือน (Status Triggers)

  • Ratio < 1.20 : ⚠️ ต่ำกว่าเกณฑ์ - Inverter ใหญ่เกินไป สิ้นเปลืองงบประมาณ ควรเพิ่มแผง
  • 1.20 ≤ Ratio ≤ 1.40 : ✅ ปกติ (Optimal) - เป็นช่วงที่ วสท. และผู้ผลิตแนะนำว่าคุ้มค่าที่สุด
  • Ratio > 1.40 : 🔴 สูงกว่าเกณฑ์ - แผงเยอะเกินไป Inverter จะตัดการทำงานทิ้งในช่วงเที่ยง (Clipping) เสียพลังงานทิ้งฟรี

19. การคำนวณขนาดท่อร้อยสายและรางเคเบิล (Conduit & Wireway Sizing)

อ้างอิงมาตรฐาน วสท. หมวดที่ 5 ว่าด้วยเรื่องการติดตั้งระบบท่อและรางสายไฟ เพื่อป้องกันความร้อนสะสมและการลัดวงจร การร้อยสายไฟ (PV Cable) ลงในท่อหรือรางจะต้องมีพื้นที่ว่าง (Free Space) เพื่อระบายอากาศตามกฎเกณฑ์ดังต่อไปนี้

19.1 การคำนวณพื้นที่หน้าตัดของสายไฟ (Cable Area)

ผู้พัฒนาต้องมีฐานข้อมูล (Lookup Table) ของเส้นผ่านศูนย์กลางภายนอก (Overall Diameter: OD) ของสาย PV Cable แต่ละขนาด (เช่น 4 sq.mm, 6 sq.mm) รวมฉนวนแล้ว

// สูตรหาพื้นที่หน้าตัดสายไฟ 1 เส้น (ตารางมิลลิเมตร)
Area_1_Cable = π * (OD / 2)2

// พื้นที่รวมของสายไฟทั้งหมดในท่อ/ราง
Total_Cable_Area = Area_1_Cable * จำนวนสายไฟทั้งหมด

19.2 ขนาดท่อร้อยสายไฟ (Conduit Sizing)

วสท. บังคับว่า กรณีร้อยสายไฟตั้งแต่ 3 เส้นขึ้นไปในท่อเดียวกัน พื้นที่หน้าตัดรวมของสายไฟต้อง ไม่เกิน 40% ของพื้นที่หน้าตัดภายในของท่อ (Internal Area)

// การหาพื้นที่ภายในท่อขั้นต่ำที่ต้องการ
Min_Conduit_Internal_Area = Total_Cable_Area / 0.40;

// นำค่า Min_Conduit_Internal_Area ไปเทียบกับตารางสเปกท่อ (เช่น ท่อ EMT/IMC/UPVC ขนาด 1/2", 3/4", 1")
// เลือกท่อขนาดถัดไปที่ใหญ่กว่าค่า Min_Conduit_Internal_Area

19.3 ขนาดรางเดินสายไฟ (Wireway / Cable Tray Sizing)

วสท. บังคับว่า สำหรับรางเดินสายไฟแบบปิดมิดชิด (Wireway) พื้นที่หน้าตัดรวมของสายไฟและฉนวน ต้อง ไม่เกิน 20% ของพื้นที่หน้าตัดภายในของราง

// การหาพื้นที่ภายในรางขั้นต่ำที่ต้องการ
Min_Wireway_Area = Total_Cable_Area / 0.20;

// นำค่า Min_Wireway_Area ไปคำนวณหากว้างxสูง ของราง (กว้าง * ลึก)
// ตัวอย่าง: ราง 50x50 mm มี Area = 2500 sq.mm

หมายเหตุ: หากเป็นระบบโรงงานที่ใช้ รางเคเบิลแบบระบายอากาศ (Cable Ladder / Perforated Tray) ข้อกำหนดการจัดเรียงสายไฟจะเปลี่ยนไปเป็นการเรียงชั้นเดียว (Single Layer) ชิดกันหรือเว้นระยะตามพิกัดกระแส (Derating Factor) อ้างอิงตามภาคผนวก ฌ. ของ วสท.
